Intro

Javascript was initially created to “make web pages come alive”.

It was developed by Brenden Eich in 1995. He was a Netscape Communications Corporation programmer & it took only 10 days to create it & was named Mocha. Later the name got changed to Livescript and then finally to JavaScript as we know it today.

The popularity of JavaScript grew & in the year 1996 it was handed over to the international standards scripting language body called ECMA (European Computer Manufacturers Association), which is responsible for the development and upkeep of the language to this day.

JavaScript was used to add programs to web pages in the browser. The language was adopted by all other major web browsers. Since then it has made modern web applications possible—applications with which you can interact directly without doing a page reload for every action.

Here we will be discussing about the data types used in javascript-

It is broadly classified in to Primitive and Advanced data types.

Primitive data types

  • String - A sequence of characters that represent a text value within quotes. For e.g: "Hello!".
  • Number - An integer or floating point number. For e.g: 42 or 3.14159.
  • Boolean - true and false.
  • undefined - If a variable is only declared & no value is assigned then it will show as undefined.
  • Null - A special keyword denoting a null value, here we assign as null or no value.

Advanced data types

  • Objects - Objects are used to store collections of data and more complex entities.
  • Symbols - The symbol type is used to create unique identifiers for objects.

String

A String in JavaScript is any letter, word or sentences.


They must be written within quotes.

let str = "Hello";

let str2 = 'Single quotes are ok too';

let phrase = `can embed another ${str}`;

In JavaScript, there are 3 types of quotes.

Double quotes: "Hello".
Single quotes: 'Hello'.
Backticks: `Hello`.

Double and single quotes are “simple” quotes. There’s practically no difference between them in JavaScript.

Backticks are “extended functionality” quotes. They allow us to embed variables and expressions into a string by wrapping them in ${…}, for example:

alert( `Hello, ${name}!` ); //the result is: Hello, John!

Numbers

In Javascript Numbers are any number- whole or decimal.

In modern JavaScript, there are two types of numbers:Regular & BigInt.

The number type represents both integer and floating point numbers.


Regular numbers in JavaScript are stored in 64-bit format IEEE-754, also known as “double precision floating point numbers”. These are numbers that we’re using most of the time, and we’ll talk about them in this chapter.

for e.g:

let n = 123; n = 12.345;

There are many operations for numbers, e.g. multiplication *, division /, addition +, subtraction -, and so on.

BigInt numbers, to represent integers of arbitrary length. They are sometimes needed, because a regular number can’t exceed 253 or be less than -253. As bigints are used in few special areas, we devote them a special chapter BigInt.

BigInt type was recently added to the language to represent integers of arbitrary length.

A BigInt value is created by appending n to the end of an integer:

// the "n" at the end means it's a BigInt const bigInt = 1234567890123456789012345678901234567890n;

Boolean

The boolean type has only two values: true and false

This type is commonly used to store yes/no values: true means “yes, correct”, and false means “no, incorrect”.

For instance:

let nameFieldChecked = true; // yes, name field is checked
let ageFieldChecked = false; // no, age field is not checked

Boolean values also come as a result of comparisons:

let isGreater = 4 > 1; alert( isGreater ); // true (the comparison result is "yes")

In JavaScript, the operator is a little bit trickier and more powerful. But first, let’s see what happens with boolean values.

There are four possible logical combinations:

alert( true || true ); // true
alert( false || true ); // true
alert( true || false ); // true
alert( false || false ); // false



Undefined

The meaning of Undefined is “value is not assigned”.

If a variable is declared, but not assigned, then its value is undefined:

let age; alert(age); // shows "undefined"

Technically, it is possible to explicitly assign undefined to a variable:

let age = 100; // change the value to undefined age = undefined; alert(age); // "undefined"

undefined means “no value.” Uninitialized variables are undefined:

> var foo; > foo undefined

Missing parameters are undefined:

> function f(x) { return x } > f() undefined

If you read a nonexistent property, you get undefined:

> var obj = {}; // empty object > obj.foo undefined

…But we don’t recommend doing that. Normally, one uses null to assign an “empty” or “unknown” value to a variable, while undefined is reserved as a default initial value for unassigned things.

Null

Null means “no object.” It is used as a nonvalue whenever an object is expected.

The special null value does not belong to any of the types described above.

It forms a separate type of its own which contains only the null value:

let age = null;

In JavaScript, null is not a “reference to a non-existing object” or a “null pointer” like in some other languages.

It’s just a special value which represents “nothing”, “empty” or “value unknown”.

The code above states that age is unknown.

typeof null // "object" (2)

The result of typeof null is "object". That’s an officially recognized error in typeof behavior, coming from the early days of JavaScript and kept for compatibility. Definitely, null is not an object. It is a special value with a separate type of its own.
